# 金融防詐AI與可信任治理比較

最後更新：2026-04-17

## 比較範圍

- 來源 A：[[來源/使用AI於金融防詐]]
- 來源 B：[[來源/在生成式浪潮時代打造值得信任的AI]]

## 對照表

| 面向 | 使用AI於金融防詐 | 在生成式浪潮時代打造值得信任的AI |
|---|---|---|
| 主要目標 | 提升金融詐騙偵測與攔阻成效 | 降低生成式 AI 導入風險，建立可被信任的運作機制 |
| 問題層級 | 業務應用層（防詐情境） | 治理與資安層（跨場景） |
| 核心方法 | 圖譜分析（community/centrality）+ XGBoost + 行為特徵 | OWASP LLM Top 10 + Guardrail + 部署/權限治理 |
| 輸出型態 | 可疑帳戶辨識、車手偵測、攔阻成效 | 風險分類、控制措施、導入治理藍圖 |
| 主要風險 | 誤報/漏報、資料覆蓋不足、跨機構資料整合困難 | Prompt Injection、Hallucination、Jailbreaking、Data Poisoning |
| 成效衡量 | Precision、識別帳戶數、避免損失金額 | 風險事件下降、誤用率下降、稽核通過率與控制覆蓋率 |
| 導入關鍵 | 資料品質、圖譜建模、人工覆核流程 | 風險優先級、控制基線、部署策略（雲/地/混合） |

## 關係與整合結論

1. 兩者是上下游關係：
`金融防詐 AI` 負責創造成效，`可信任治理` 負責讓成效可長期、安全、可稽核地維持。
2. 只做模型不做治理，容易在資安與法遵上失守；只做治理不做模型，則難以產生實質防詐成效。
3. 最佳做法是雙軌推進：
一軌優化偵測能力（模型與特徵），一軌建立治理能力（風險控制與監控）。

## 建議落地順序

1. 先定義防詐 KPI：Precision、召回、攔阻金額、人工覆核成本。
2. 依 OWASP 風險盤點最低控制基線（輸入過濾、輸出檢查、權限隔離、審計紀錄）。
3. 在單一業務場景試點，建立「模型成效 + 治理指標」雙儀表板。
4. 再擴展到跨團隊與跨資料域。
